# Codex Creative Direction Audit

Date: 2026-04-06

This is a Codex-owned assessment of the current Ashworth Manor asset base and
authoring direction. It is intentionally separate from any prior agent notes.

## Executive Judgment

Ashworth Manor currently has **more than enough assets** to make a strong game.

The problem is not scarcity. The problem is **authoring bias**.

The project has been leaning too hard on imported models as the default answer,
when the actual needs of the game point toward:

- procedural-first architecture
- procedural or hybrid mechanisms
- shape-kit setpieces
- material-driven identity
- selective model use where silhouette truly matters

This should not become dogma in the opposite direction. Some things absolutely
benefit from models. The right rule is:

- let the needs of the player experience decide
- let the silhouette decide
- let animation/state requirements decide

## Fonts

Local font inventory is strong and already thematically correct:

- Cinzel family for titles / ritual / brass plaque / menu marker use
- Cormorant family for body text / documents / intimate reading

Judgment:

- no new font hunt is needed right now
- typography is already one of the strongest, most coherent parts of the project
- we should lean harder into it for diegetic labels, signage, plaques, ledgers, and printed matter

## Audio

Local audio inventory is broader than the game is currently using:

- loop library is strong for tone and room mood
- tension layers are already organized by zone/function
- footstep surfaces are better covered than the current game presentation suggests
- SFX categories exist, but the player-facing tactile vocabulary still needs expansion in use

Judgment:

- ambient and footstep foundations are good
- the biggest audio opportunity is **material tactility**, not more ambient music

Highest-value missing-feel categories in active runtime use:

- chain rattle
- brass plaque tap / rub
- iron gate pressure and hinge complaint
- wood sign creak
- carpeted step dampening
- greenhouse glass touch / knock
- ceramic / porcelain contact
- marble / stone scrape
- hidden seam click / hollow-wall response

## Local Shared Models

The local project asset tree has solid decorative coverage:

- chandeliers
- fireplaces
- candle holders
- statues
- rugs
- picture frames / blanks

Judgment:

- these are good as silhouette anchors and environmental dressing
- they should remain mostly model-first
- they should not dictate the architecture system

Good model-first candidates:

- porcelain doll
- statuary
- luggage
- chandeliers
- complex decorative frames
- high-character furniture silhouettes

## Texture Inventory

The texture situation is better than the runtime is currently expressing.

There are enough textures to support a strong estate material language without
requiring bespoke modeled versions of every object.

Judgment:

- textures should be doing far more of the work
- the current direction should shift toward reusable material families

Priority material families to standardize:

- brass
- blackened iron / chain iron
- varnished oak / walnut / mahogany
- painted moulding wood
- marble
- porcelain / ceramic glaze
- Persian / runner textile
- frosted and clear glass
- slate / stone / gravel

## 3DPSX Library Assessment

Mounted library review confirms a very large PSX inventory under `/Volumes/home/assets/3DPSX`.

Strong categories for Ashworth Manor:

- `Environment/Nature`
  - hedges
  - fences
  - tree packs
  - rocks
  - maze/estate boundary support
- `PSX Mega Pack II`
  - doors and gates
  - light sources
  - debris
  - modular structures
  - utility props
- `Fantasy/Books and Scrolls`
  - readable clue dressing
- `Fantasy/Bottles`
  - pantry, lab, cellar, greenhouse, chapel dressing
- `Props/Farm`
  - carriage-house yard, rear estate, greenhouse service edge, posts, carts, pumps, notice boards

Weak or low-priority categories for Ashworth Manor:

- weapons
- combat-heavy props
- overt gore/flesh packs except in very controlled horror beats
- modern/electrical packs unless carefully filtered

Judgment:

- the library is especially strong for grounds, outbuildings, boundaries, garden service clutter, gates, and rural estate dressing
- it is less compelling as a one-to-one source for every refined interior object
- it should be treated as a **silhouette and dressing library**, not as the architecture engine

## Procedural vs Model Determinations

### Should bias procedural or hybrid

- walls
- floors
- ceilings
- doors
- windows
- gates
- stairs
- ladders
- trapdoors
- tubs / basins
- tables that can be primitive-authored
- rugs
- greenhouse framing
- vessels with changing contents
- hidden seams and reveal mechanisms

### Should bias model-first

- dolls
- statues
- ornate chandeliers
- luggage
- portrait frames with carved silhouette
- very specific furniture silhouettes
- clutter piles where asymmetry matters more than interaction

### Should be hybrid

- bookcases with hidden motion
- teapots / cups / vessels
- baptismal font
- greenhouse basins
- signboards
- bathtubs with claw feet
- fireplaces with moving or lit components

## Concrete Direction

The right creative direction is:

1. keep the declaration runtime canonical
2. reduce architectural dependence on imported models
3. increase shape-kit + material-kit authoring
4. use imported models where drawing quality genuinely matters
5. judge every piece by the player’s bodily experience, not by asset availability

## Immediate Priorities

- build out the estate material vocabulary
- keep expanding the primitive shape kit
- audit room by room for “should this really be a model?”
- use PSX library assets primarily for grounds, silhouette dressing, and select hero objects
- reserve high-value models for dolls, statues, luggage, chandeliers, portrait frames, and special furniture

## Creative Director Call

My current determination is:

- Ashworth Manor should become **procedural-first, model-assisted**
- not procedural-only
- not model-driven

That balance best serves the actual game.
